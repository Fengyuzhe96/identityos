# Product Requirements Document (PRD): IdentityOS

## 1. Product Overview
**Name:** IdentityOS
**Tagline:** Кибернетический сдвиг идентичности.
**Vision:** Переход от поверхностного отслеживания привычек (habit tracking) к фундаментальной трансформации личности через принципы кибернетики и геймификации. Приложение помогает пользователям сломать старые паттерны поведения ("автопилот") и выстроить новую идентичность.

## 2. Core Modules & Mechanics (Уже реализовано)

### 2.1. Onboarding ("Психологические раскопки")
Глубокий протокол инициализации, где пользователь определяет базовые векторы своего развития:
- **Anti-Vision (Точка диссонанса):** Описание нежелательного будущего, которого нужно избежать.
- **Vision (Конструирование):** Образ идеального будущего.
- **Year Goal (Игровой Сезон):** Главная цель на 1 год.
- **Month Project (Битва с боссом):** Интенсивный фокус на 30 дней.
- **Constraints (Правила Игры):** Одно жесткое ограничение (например, "никаких соцсетей до полудня").
- **Daily Levers (Ежедневные рычаги):** 2-3 ключевых действия, которые генерируют максимальный результат. Сущности хранятся с UUID уникальными идентификаторами.

### 2.2. Система "Линз" (Lens Toggle)
Бесшовное переключение между двумя режимами восприятия:
- **Macro (Стратегия):** Обзор Vision, Anti-Vision, глобальных целей и ограничений. Дает понимание "Зачем я это делаю".
- **Micro (Тактика):** Список ежедневных квестов (Daily Levers). Фокус на том, "Что мне нужно сделать сегодня".

### 2.3. Forcefield Mode (Режим глубокого фокуса)
Полноэкранный иммерсивный режим для выполнения одной задачи за раз.
- Устраняет все визуальные раздражители.
- Использует гипнотические "дышащие" анимации (Framer Motion) для удержания концентрации.
- Выход из режима происходит только по факту выполнения выбранного квеста.

### 2.4. Кибернетический цикл и Сенсорика Системы
После каждого выполненного квеста система запрашивает оценку состояния пользователя:
- **Alive (Живой):** Действие наполнило энергией и приблизило к Vision (+20 XP).
- **Dead (Мертвый):** Действие вызвало сопротивление или отдалило от цели (Коррекция, +5 XP).
*Эта механика заменяет бездумное проставление галочек на осознанную рефлексию.*

### 2.5. Дневник Состояний (State Diary) и Аналитика
Визуализация истории кибернетических обратных связей:
- **30-дневная статистика:** Доля состояний "Живой" (Виталити) против "Мертвый".
- **Мини-график (Bar chart):** Ежедневная агрегация количества квестов и соотношения состояний.
- **История записей:** Хронометраж выполненных рычагов с указанием полученного опыта.

### 2.6. Эволюция Эго (Система Прокачки)
Геймификация через уровни осознанности:
- Уровень 1: Конформист (Автопилот)
- Уровень 2: Искатель (Осознание)
- Уровень 3: Архитектор (Проектирование)
- Уровень 4: Стратег (Управление)
- Уровень 5: Творец (Идентичность)
Переход на новый уровень осуществляется по мере накопления очков опыта (XP).

### 2.7. Pattern Breaker (Прерыватель автопилота)
Модуль, имитирующий случайные "пинг"-уведомления от системы, цель которых — вырвать пользователя из состояния бездумного скроллинга и вернуть фокус на реальность.

## 3. Техническая Архитектура & Стек

### Фронтенд:
- **Framework:** Next.js 15 (App Router)
- **Language:** TypeScript
- **Styling:** Tailwind CSS v4
- **Animations:** Framer Motion (физические программируемые анимации — spring, easeInOut, layout transitions)
- **State Management:** Zustand (Глобальный `useIdentityStore`)
- **Icons:** Lucide React
- **Typography:** Geist & Geist Mono

### Бэкенд & База данных (Supabase):
- **Auth:** Email + Password, интеграция Google OAuth.
- **Database:** PostgreSQL (3 основные таблицы: `profiles`, `daily_levers`, `state_diary`).
- **Data Fetching:** `@supabase/ssr` клиентские и серверные утилиты с поддержкой middleware-защиты роутов.
- **Security:** Настроены Row Level Security (RLS) политики (каждый пользователь имеет доступ только к своим данным).
- **Sync Architecture:** Debounce-синхронизация локального `Zustand` хранилища с облачной базой данных. Внедрен механизм миграции данных из локального хранилища (localStorage) в Supabase при первом логине.

## 4. Дизайн и UX/UI Эстетика
- **Цветовая палитра:** Глубокая темная тема (Zinc-950/Black), белые акценты для фокуса, неоновые статусные цвета (Red/Green). Воздержание от визуального шума.
- **Анимации:** Дышащие, "жидкие" переходы (motion blur при смене фаз, spring effects для модалок). 
- **Эстетика:** "Анти-дофаминовая" премиальная атмосфера в стиле футуристичного интерфейса операционной системы сознания.

## 5. Ближайшие планы (Roadmap / Ожидающие фичи)
1. **Модуль "Синтеза" и Рефлексии:** Фрирайтинг / Brain Dump сессии по утрам или при срабатывании Pattern Breaker.
2. **Оценка ценности рычагов (Leverage):** Система приоритезации `High/Medium/Low Leverage` при создании новых ежедневных квестов.
3. **Виртуальное Дерево Навыков (Skill Tree):** Визуализация получения перков за выполнение "Битвы с Боссом" (проектов месяца).
